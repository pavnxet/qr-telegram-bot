# 🤖 QR Code Generator Telegram Bot (Cloudflare Workers)

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Deploy to Cloudflare Workers](https://img.shields.io/badge/Deploy-Cloudflare-f38020?logo=cloudflare)](https://dash.cloudflare.com/)
[![Telegram Bot](https://img.shields.io/badge/Telegram-Bot-blue?logo=telegram)](https://t.me/botfather)

A fast, serverless Telegram bot that instantly converts any text into a QR code image. Built on **Cloudflare Workers** (JavaScript) with **Workers KV** for persistent statistics and rate limiting.

- **Instant QR Generation**: Send any text and receive a QR code image in seconds.
- **Multi‑Provider Fallback**: Uses `goQR.me` with automatic fallback to `QuickChart.io`.
- **Customizable Size**: Use `/qr small|medium|large <text>` to control QR size.
- **Inline Keyboard**: Quick‑access buttons for stats and generating another code.
- **Usage Tracking**: Global and per‑user counters stored in KV.
- **Rate Limiting**: Prevents abuse (5 requests per 10 seconds per user).
- **100% Free**: Runs within Cloudflare's generous free tier.

## 🚀 Deployment (Cloudflare Dashboard)

This is the easiest method – no terminal required.

### Step 1: Create a KV Namespace
1. Log in to [Cloudflare Dashboard](https://dash.cloudflare.com/).
2. Navigate to **Workers & Pages** → **KV**.
3. Click **Create namespace**.
4. Name it `QR_BOT_STATE` (or anything you like).
5. Copy the **ID** that appears – you’ll need it later.

### Step 2: Create a Worker
1. Go to **Workers & Pages** → **Overview** → **Create application** → **Create Worker**.
2. Give your worker a name, e.g., `qr-telegram-bot`.
3. Click **Deploy** (you can leave the placeholder code for now).

### Step 3: Replace the Code
1. After deployment, click **Edit code**.
2. Delete all existing code.
3. Copy the **entire** [`worker.js`](./worker.js) from this repository and paste it into the editor.
4. Click **Save and Deploy**.

### Step 4: Set Environment Variable
1. In the worker’s page, go to **Settings** → **Variables**.
2. Under **Environment Variables**, click **Add variable**.
3. Enter:
   - **Variable name:** `TELEGRAM_BOT_TOKEN`
   - **Value:** `123456789:ABCDefGhiJkLmNopQRstuVwXyz` *(your actual token)*
4. Click **Save**.

### Step 5: Bind the KV Namespace
1. Still in **Settings** → **Variables**, scroll to **KV Namespace Bindings**.
2. Click **Add binding**.
3. Configure:
   - **Variable name:** `BOT_STATE` *(must match the code)*
   - **KV namespace:** Select the namespace you created (`QR_BOT_STATE`).
4. Click **Save**.

### Step 6: Activate the Webhook
1. Find your worker’s URL (e.g., `https://qr-telegram-bot.<your-subdomain>.workers.dev`).
2. Open a browser tab and visit:
   ```
   https://<YOUR_WORKER_URL>/set-webhook
   ```
3. You should see a JSON response with `"ok": true`. This tells Telegram where to send updates.

### 🎉 Done!
Open Telegram, search for your bot, and send `/start`. You’re ready to generate QR codes!

## 💻 Deployment (Wrangler CLI) – Optional

For local development or automated deployments.

1. **Clone the repository**
   ```bash
   git clone https://github.com/yourusername/qr-telegram-bot.git
   cd qr-telegram-bot
   ```

2. **Install Wrangler**
   ```bash
   npm install -g wrangler
   ```

3. **Authenticate with Cloudflare**
   ```bash
   wrangler login
   ```

4. **Create `wrangler.toml`**
   ```toml
   name = "qr-telegram-bot"
   main = "worker.js"
   compatibility_date = "2026-04-15"

   kv_namespaces = [
     { binding = "BOT_STATE", id = "your-kv-namespace-id" }
   ]

   [vars]
   # TELEGRAM_BOT_TOKEN will be set as a secret

   [triggers]
   crons = []   # No cron needed for this bot
   ```

5. **Set the secret**
   ```bash
   wrangler secret put TELEGRAM_BOT_TOKEN
   ```

6. **Deploy**
   ```bash
   wrangler deploy
   ```

7. **Set the webhook**
   ```bash
   curl "https://<YOUR_WORKER_URL>/set-webhook"
   ```

## 📱 Usage

### Basic QR Generation
Simply send any text message (that isn’t a command) to the bot. It will reply with a QR code image.

**Example:**
```
Hello, World!
```
*Bot replies with a QR code containing "Hello, World!"*

### Custom Size
Use the `/qr` command followed by a size keyword and your text.

| Command | Size |
|---------|------|
| `/qr small Hello` | 150×150 px |
| `/qr medium Hello` | 300×300 px (default) |
| `/qr large Hello` | 500×500 px |

### Inline Keyboard
After receiving a QR code, you’ll see buttons:
- **📊 My Stats** – Shows how many QR codes *you* have generated.
- **🌍 Global Stats** – Shows total QR codes generated by all users.
- **🔄 Generate Another** – Prompts you to send new text.

## ⚙️ Customization

You can easily tweak the bot’s behavior by editing `worker.js`.

### Change QR Providers
Edit the `QR_APIS` array at the top of the file. Each provider must have a `name` and a `url` function that returns the API endpoint.

### Adjust Rate Limits
Modify the `checkRateLimit` function. The current window is 10 seconds with a max of 5 requests:
```javascript
const windowStart = now - 10000;  // 10 seconds
if (timestamps.length >= 5) { ... }
```

### Change Default QR Size
In the `parseCommand` function, update the `sizeMap`:
```javascript
const sizeMap = { small: 150, medium: 300, large: 500 };
```

### Disable Inline Keyboard
Comment out or remove the `reply_markup` property in the `sendPhoto` call.

## 🔧 Troubleshooting

### Bot doesn’t respond to messages
- Ensure the webhook is set correctly: visit `https://<YOUR_WORKER_URL>/set-webhook` and check the JSON response.
- Check the **Logs** tab in the Cloudflare Dashboard for errors.
- Verify your `TELEGRAM_BOT_TOKEN` is set correctly in environment variables.

### "Failed to send QR image" error
- The text might contain characters that break Telegram’s Markdown parser. The bot has a fallback to plain text; if that also fails, check the logs for the specific API error.
- Try a shorter text (QR APIs have limits, usually ~2,000 characters).

### Rate limit messages
- You’re generating QR codes too quickly. Wait a few seconds and try again.
- To disable rate limiting, remove the `checkRateLimit` call in the message handler.

### QR code looks blurry
- Use `/qr large <text>` to get a higher resolution image.
- The external API may be returning a smaller image; check the provider’s documentation.
